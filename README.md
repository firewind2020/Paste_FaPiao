# 贴发票 功能说明

## 项目概述
本工具用于处理 4 类单页 PDF 发票：住宿、餐饮、高铁、飞机。

核心能力：
1. 自动识别发票类型并提取关键信息。
2. 按“发生日期”升序统一排序。
3. 导出 Excel 明细。
4. 生成 Word 粘贴版（每页 2 张），并在末尾追加专票二次粘贴区。

## 处理流程
1. 提取并统计发票信息。
2. 按“发生日期”升序排序。
3. 将结果写入 Excel。
4. 按排序结果将 PDF 发票排版到 Word（每页 2 张）。
5. 首轮插入完成后，在 Word 末尾插入换页符。
6. 将所有“专票”按“发生日期”升序再次插入（同样每页 2 张）。

## 运行方式
```bash
python paste_fapiao.py --input-dir .
```

### 参数说明
| 参数 | 说明 | 默认值 |
| --- | --- | --- |
| `--output` | 指定 Excel 输出文件名 | 发票信息提取结果.xlsx |
| `--word-output` | 指定 Word 输出文件名 | 发票粘贴.docx |
| `--skip-word` | 只导出 Excel，不生成 Word | 关闭 |

### 默认输出
1. Excel：发票信息提取结果.xlsx
2. Word：发票粘贴.docx

2. pdfplumber

4. python-docx

## 发票分类方法（已实现）

### 高铁
满足任一条件即判定为高铁：
1. 发票首行包含“电子客票”。
2. 全文包含“电子客票”（兼容 OCR 换行）。
3. 同时命中高铁结构特征：车次模式（如 G27）+ 发车时间模式（如 YYYY年MM月DD日 HH:MM开）。

### 餐饮
项目名称区域（优先）或全文包含“餐饮”。

### 住宿
项目名称区域（优先）或全文包含“住宿”。

### 飞机
满足任一条件即判定为飞机：
1. 项目名称区域包含“机票”。
2. 销售方名称含“航空”且项目名称区域包含“运输服务”。
3. 项目名称区域包含“民航发展基金”。
4. 全文兜底命中“机票”或“民航发展基金”，或“运输服务 + 航空”。

## 信息提取规则

### 统一提取字段
文件名、发票类型、是否专票、开票日期、发生日期、发票号码、购买方名称、发票金额、销售方名称。

### 是否专票判断
读取发票首个非空行：
1. 包含“专用发票” -> 是否专票 = 是。
2. 否则 -> 是否专票 = 否。

### 发生日期规则
1. 住宿：发生日期 = 开票日期。
2. 高铁：发生日期 = 发车日期。
3. 餐饮：发生日期 = 开票日期。
4. 飞机：从备注/航程相关文本提取，优先级如下：
   - 模式 A：`YYYY/MM/DD`
   - 模式 B：`MM月DD日`（自动补当前年份；若出现“开票日期早于发生日期”，则发生日期年份减一，用于跨年出行次年开票场景）
   - 若仍未提取到：回退为开票日期（避免空值）

### 导出排序规则
按“发生日期”升序排序；无法解析的日期排在最后：
1. Excel 导出按该顺序写入。
2. Word 首轮粘贴按该顺序入页。
3. Word 末尾换页后，专票二次粘贴也按该顺序入页。

### 住宿天数提取规则
按以下优先级提取：
1. 从文件名日期区间推算（例如 `2025-12-05~06 -> 2`）。
2. 从住宿明细行中提取数量列（并过滤异常值，避免把单价/金额识别为天数）。
3. 兜底：从正文“入住/离店日期”推算住宿天数。

### 高铁额外提取
始发站、终到站、车次、发车日期、发车时间、车厢座位号、乘客姓名。

### 高铁销售方规则
当发票类型为高铁时，销售方名称固定为“中国国家铁路集团”。

### 飞机额外提取
航班号、始发城市、目的城市。

#### 始发城市、目的城市提取算法
参考下文“中国大陆的民用定期航班通航城市”名单，在备注/航程文本中匹配“城市-城市”（或“城市—城市”“城市至城市”），第一个城市为始发地，第二个城市为目的地。

#### 航班号提取算法
参考下文“中国国内航空公司的 IATA 航班代码”，在备注/航程文本中匹配“2 位航司代码 + 3 或 4 位数字”作为航班号。
